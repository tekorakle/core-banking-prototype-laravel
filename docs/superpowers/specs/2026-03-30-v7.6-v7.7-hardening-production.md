## v7.6.0 — Security & CI Hardening

### Threat Model Remediation (13 remaining findings)

Findings #1 (JIT TOCTOU) and #2 (Webhook SSRF) were fixed in v7.1.1. Remaining:

| # | Finding | Severity | Fix |
|---|---------|----------|-----|
| 3 | Bridge quote not stored server-side | High | Add server-side quote cache in BridgeOrchestratorService — store quotes in Cache with 60s TTL, client sends quote_id only |
| 4 | ZK private inputs in temp files not securely deleted | High | Wrap SnarkjsProverService proof generation in try/finally with `@unlink()` cleanup |
| 5 | Open Banking webhook uses session-based state | High | Replace session state with database-persisted nonce (create `openbanking_states` table or use Cache with 10-minute TTL) |
| 6 | No concurrency limit on ZK proof generation | High | Add semaphore via `Cache::lock('zk_proof_semaphore', 120)` with configurable max concurrent proofs |
| 7 | `$allowWithoutTenant` publicly writable | Medium | Make property private, add controlled setter with logging |
| 8 | Coinbase/Paysera webhooks lack replay protection | Medium | Track delivery IDs in cache — reject duplicates within 24h window |
| 9 | No per-card-token rate limiting | Medium | Add card-level rate limiter in JitFundingService using `RateLimiter::attempt("jit:{cardToken}", 10, fn() => true, 60)` |
| 10 | Bridge sender address not verified | Medium | Add wallet ownership check in BridgeOrchestratorService — verify sender address belongs to authenticated user via BlockchainAddress model |
| 11 | No HTTPS enforcement on outbound webhooks | Medium | Add HTTPS check in WebhookService — reject non-HTTPS URLs in production |
| 12 | Outbound webhook payloads not sanitized | Medium | Add PayloadSanitizer that strips fields matching patterns: *_secret, *_key, *_token, password, ssn, pan |
| 13 | Circuit file integrity not verified | Medium | Add SHA-256 hash manifest file for circuits — verify before proof generation |
| 14 | No bridge value/frequency limits | Low | Add configurable bridge limits in config/crosschain.php, enforce in BridgeOrchestratorService |
| 15 | Marqeta HMAC optional | Low | Log warning when HMAC secret not configured in production, add to health check |
